# PoE2 工具箱

PoE2 工具箱是一个用于《流放之路 2》集市页面的 Chrome 扩展。它会在交易网页右侧加入一个侧边栏，把搜索历史、收藏管理、复制 PoB、查看词缀和妄想症统计集中到一个地方。

这个项目适合经常在 PoE2 集市筛选装备、保存搜索条件、复制物品到 Path of Building，或需要批量查看 Megalomaniac 天赋珠词条的玩家。

## 主要功能

| 功能 | 能做什么 | 使用位置 |
| --- | --- | --- |
| 搜索历史 | 自动记录最近的集市搜索，点击记录可回到当时的搜索页 | 侧边栏 tab |
| 收藏管理 | 保存常用搜索条件，支持文件夹、拖拽整理、导入导出 | 侧边栏 tab |
| 复制 PoB | 在搜索结果行加入「复制PoB」按钮，复制 Path of Building 可导入的英文物品文本 | 搜索结果行 |
| 查看词缀 | 根据当前选择的物品类型，从 poe2db 查看可出现的词缀 | 侧边栏 tab |
| 妄想症统计 | 输入 poe.ninja 构筑列表链接，统计 Megalomaniac 天赋词条出现次数，并生成购买链接 | 侧边栏 tab |

## 支持页面

当前扩展会加载在以下 PoE2 集市页面：

- 国服：`https://poe.game.qq.com/trade2*`
- 国际服：`https://www.pathofexile.com/trade2*`

国服功能默认开启。国际服功能默认关闭，需要在扩展设置页中手动开启。

搜索历史、收藏、妄想症统计结果等用户数据会按服务器分别保存，国服和国际服互不混用。

## 安装方式

这个扩展目前以“加载已解压扩展”的方式使用。

1. 打开 Chrome 的 `chrome://extensions`。
2. 打开右上角「开发者模式」。
3. 点击「加载已解压的扩展程序」。
4. 选择本项目目录 `poe2-toolbox/`。
5. 打开 PoE2 集市页面，例如 `https://poe.game.qq.com/trade2/`。

页面右侧出现工具箱侧边栏后，即表示加载成功。点击浏览器工具栏里的扩展图标可以开合侧边栏。

## 基本使用

### 开关功能

右键扩展图标进入「选项」，或在侧边栏点击设置按钮，可以分别控制国服和国际服的功能开关。

国服默认启用所有功能。国际服由于测试覆盖较少，默认全部关闭。

### 搜索历史

在集市页面提交搜索、点击搜索按钮、URL 变化或首次进入搜索页时，扩展会尝试记录当前搜索条件。

在「搜索历史」tab 中可以：

- 查看最近的搜索记录。
- 点击记录回到对应搜索页面。
- 删除单条记录。
- 清空全部历史。

### 收藏管理

在「收藏」tab 中可以把当前搜索条件保存下来，适合保存常用装备、珠宝、通货或过滤条件。

收藏支持：

- 收藏当前搜索。
- 新建文件夹。
- 将收藏拖入文件夹。
- 重命名或删除文件夹。
- 导出文件夹为压缩文本。
- 从导出文本导入文件夹。

### 复制 PoB

在 PoE2 集市搜索结果中，每条物品结果旁会出现「复制PoB」按钮。点击后，扩展会把该物品转换为 Path of Building 可以识别的英文文本，并写入剪贴板。

第一次使用时需要下载 PoB 翻译词典原料，体积约 5.7 MB。扩展会在本地生成并缓存词典，后续会复用缓存，不需要每次重新下载。

妄想症、妄想之眼等「Allocates」类珠宝的核心天赋行（如「配置 [fire58|伊柯洛塔的狱火]」）会被翻译成 PoB 标准的「Allocates <英文名>」。核心天赋英文名表从 poe2db 英文页天赋树 JSON 单独拉取并缓存，与主词典相互独立。

如果复制结果异常，可以到设置页手动更新或导入词典。

### 查看词缀

在集市页面选择物品类型后，打开「查看词缀」tab 可以查看该类型在 poe2db 上的词缀信息。

扩展会根据当前页面的筛选条件判断物品类型。某些类型如果 poe2db 没有独立词缀页，或扩展暂未配置映射，会显示暂不支持。

### 妄想症统计

「妄想症统计」用于统计 poe.ninja 构筑列表中 Megalomaniac 天赋珠上的天赋词条。

使用步骤：

1. 打开「妄想症统计」tab。
2. 点击「开始统计」。
3. 输入 poe.ninja 的 PoE2 builds 链接。
4. 输入要检查的账号数量，范围为 10 到 100。
5. 等待统计完成。
6. 点击「查看天赋详情」后，可以看到天赋 ID 和详情。
7. 勾选想搜索的天赋，选择国服或国际服，点击「去购买」生成集市搜索链接。

角色详情请求会按顺序执行，并在请求之间等待一小段时间，以降低触发 poe.ninja 限流的概率。账号数量越多，统计耗时越久。遇到 poe.ninja 429 限流时，扩展会停止后续请求并提示。

## 数据和网络说明

扩展会使用 Chrome 本地存储保存以下数据：

- 功能开关。
- 搜索历史。
- 收藏和文件夹。
- PoB 翻译词典缓存。
- 最近一次妄想症统计输入和结果。
- 天赋详情缓存。

扩展会访问以下外部站点：

| 站点 | 用途 |
| --- | --- |
| `poe2db.tw` | 查看词缀、读取天赋详情 |
| `ninja.710421059.xyz` | 下载 PoB 翻译词典原料 |
| `poe.ninja` | 读取构筑列表和角色详情，用于妄想症统计 |

除这些功能需要的本地数据和请求外，扩展没有额外的账号登录流程。

## 常见问题

### 侧边栏没有出现

先确认当前页面是否是 `trade2` 页面，并在 `chrome://extensions` 中确认扩展已启用。改动代码后需要点击扩展卡片上的「刷新」，然后重新刷新集市页面。

### 国际服为什么默认关闭

国际服功能代码已经按独立数据前缀接入，但测试覆盖少于国服。需要使用时可以在设置页手动开启。

### 复制 PoB 第一次很慢

第一次复制前需要下载词典原料并在本地建表。完成后会缓存到本地，后续会明显更快。

### 查看词缀提示不支持

通常是当前物品类型在 poe2db 上没有独立词缀页，或者扩展的类型映射表还没有覆盖该类别。

### 妄想症统计为什么耗时很长

扩展需要逐个读取 poe.ninja 角色详情，并主动降低请求频率以避免限流。统计 100 个账号会比统计 10 个账号慢很多，这是正常情况。

## 开发说明

本项目是原生 Chrome MV3 扩展，没有构建步骤，不使用打包器、TypeScript 或 lint 工具。

主要目录：

```text
content/                  页面侧边栏、按钮注入和前端功能
content/features/         各个功能模块
background/               后台 service worker 和跨域请求
lib/                      PoB 词典归一化、建表和压缩库
options/                  扩展设置页和功能介绍页
tools/                    本地验证和调试脚本
```

改完 JS 后建议执行语法检查：

```bash
node --check content/features/example.js
```

修改 PoB 词典归一化或建表逻辑后，需要跑正确性校验：

```bash
node tools/verify-dict.mjs
```

本地调试妄想症统计：

```bash
node tools/poe-ninja-megalomaniac.mjs "<poe.ninja builds 链接>" 20
```

开发时修改文件后，需要在 `chrome://extensions` 中刷新扩展。content script 改动还需要刷新目标集市页面。
